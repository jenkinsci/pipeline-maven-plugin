Seuil pour déclencher les constructions avals, basé sur la phase atteinte du cycle de vie Maven pendant l'exécution Maven.

Si l'option "install" est selectionnée, alors les constructions avals seront déclenchées lors d'un "mvn clean install" ou "mvn clean deploy" mais pas lors d'un "mvn clean verify" ou "mvn clean package".

Exemple

Configurer un projet Jenkins multi-branches avec

  • Seuil : "deploy"
  • exécution de "mvn clean deploy" sur les branches (incluant master) et exécution de "mvn clean verify sur les demandes d'intégration
Ce qui donnera :
  • Les constructions des branches (incluant "master") vont générer des fichiers jar/war qui seront envoyés dans le dépôt Maven d'entreprise et déclencheront les constructions avals
  • Les constructions des demandes d'intégration vont uniquement générer le livrable mais qui ne sera pas envoyé dans le dépôt Maven, et les constructions avals ne seront donc pas déclenchées
