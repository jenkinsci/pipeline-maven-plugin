Lorsqu'actif, toute sortie standard ou d'erreur d'une suite de test sera enregistrée dans les résultats des tests à la fin de la construction. (Ceci ne vaut que pour d'additionnels messages affichés sur la console, pas pour une stack trace.) Ces informations sont toujours sauvegardées quand le test échoue, mais par défaut les longues données d'un test réussi sont tronquées pour économiser de l'espace disque. Cocher cette option si vous voulez toujours voir les messages de log de n'importe quel test, y compris ceux réussis, mais soyez conscients que la consommation mémooire de Jenkins peut augmenter significativement en conséquence, même si vous ne regardez jamais ces résultats de test !
