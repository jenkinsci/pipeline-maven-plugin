Specifie le fichier de paramètres globaux Maven settings.xml. Le chemin précisé peut être absolu ou relatif à l'espace de travail. Le remplacement de variable d'environnement de type Shell est possible, en utilisant la syntaxe ${VARIABLE}. Le fichier sera tout d'abord recherché sur l'agent chargé de la construction. S'il n'y est pas trouvé, celui du contrôleur sera utilisé à la place. L'élément settings dans le fichier settings.xml contient les éléments utilisés pour définir des valeurs qui configurent l'exécution Maven de plusieurs façons, comme le pom.xml, mais qui ne devraient pas être embarquées dans un projet spécifique, ou distribuées publiquement. Ces valeurs incluent l'emplacement du mirroir local, un serveur distant alternatif et des identifiants de connexion.
Le fichier settings.xml peut se trouver à deux endroits par défaut :
  • L'installation Maven - par défaut: $M2_HOME/conf/settings.xml
  • Spécifique à l'utilisateur - par défaut: ${user.home}/.m2/settings.xml
Le premier est aussi appelé paramètres globaux, le second est désigné comme les paramètres spécifiques à l'utilisateur. Si les deux existent, leurs contenus est fusionné, la partie spécifique utilisateur étant prédominante.
