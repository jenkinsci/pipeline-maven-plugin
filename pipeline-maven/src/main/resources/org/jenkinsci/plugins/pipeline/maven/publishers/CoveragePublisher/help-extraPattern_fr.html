Un modèle est défini par une configuration d'inclusion d'ensemble de fichiers Ant qui définit les rapport de couverture à lire. Séparer les modèles par des espaces ou des virugles. Ces modèles sont déterminés dans l'espace de travail de Jenkins, les chemins doivent donc être obligatoirement relatifs. Si aucun modèle n'est défini, alors le paramétrage par défaut de l'outil s'appliquera automatiquement.
